# MVP-009 복귀 UX 카피 가이드

Last Updated: 2026-02-27
Owner Scope: `features/mvp/components/mvp-dashboard.tsx`

## 목적
복귀 UX(재청킹/재등록/안전 차단)에서 사용자 비난 없이, 즉시 다음 행동으로 이어지는 톤을 일관되게 유지한다.

## 카피 가이드 테이블

| 케이스 | 톤 목표 | 금지 표현(예시) | 적용 카피 | 적용 위치 |
| --- | --- | --- | --- | --- |
| 재청킹 | 부담 완화 + 즉시 실행 유도 | `왜 아직도`, `실패`, `못했네요` | `괜찮아요. 더 작은 단계로 다시 나눴어요. 첫 단계부터 이어가요.` | `handleRechunk` 피드백 |
| 재등록 | 중단 정당화 + 복귀 경로 명확화 | `포기`, `또 미룸`, `이번에도` | `괜찮아요. 내일로 다시 등록했어요. 바로 시작할 청크를 준비해뒀어요.` | `handleReschedule` 피드백 |
| 안전 차단 | 안전 우선 + 재입력 유도 | `잘못 입력`, `문제 있는`, `금지니까 안 돼요` | `괜찮아요. 안전을 위해 이 입력은 청킹하지 않았어요. 안전한 할 일로 다시 입력해 주세요.` | 위험 입력 분기 피드백 |

## CTA 라벨 원칙
- 기술 용어보다 행동 중심 문구를 우선한다.
- 복귀 경로를 버튼 텍스트에서 바로 이해할 수 있어야 한다.

| 버튼 | 적용 라벨 |
| --- | --- |
| 재청킹 CTA | `더 작게 다시 나누기` |
| 재등록 CTA | `내일로 다시 등록` |

## 점검 체크리스트
- 세 케이스 모두 `괜찮아요.`로 시작해 톤을 통일했는가
- 비난/평가/낙인형 표현이 없는가
- 사용자가 다음 행동을 바로 이해할 수 있는가
