⚠️ 비활성 문서(Deprecated / 프롬프트 사용 금지)

이 문서는 “풀데이(8시간) 템플릿” 아이디어를 정리하던 과거 메모입니다.
현재 프로젝트의 `data/validation_rules.json`(단일 기준)은 `missions` 개수를 **3~6개로 강제**합니다.
따라서 이 문서에 남아 있던 “미션을 20개+로 늘리는 풀데이 템플릿” 발상은 **현 스키마/검증 계약과 불일치**합니다.

중요
- 이 문서는 실행 지시/데이터 작성 프롬프트로 사용하지 않습니다. (복붙용 프롬프트/출력 규칙 섹션은 제거했습니다.)
- 데이터 작성은 항상 LLM이 JSON 레코드를 직접 작성(출력)하는 방식이며, 코드로 조립·치환하여 자동으로 만들어내는 방식은 금지입니다.
- 풀데이 템플릿을 실제로 도입하려면 `docs/dataset-schemas.md` 및 `data/validation_rules.json` 계약 자체를 **별도 트랙에서 확장/합의**해야 합니다. (단일 진실은 스키마/룰 문서)

메모(아이디어 수준)
- “하루 계획/휴일 플랜” 계열 입력을 별도 컨셉/클러스터로 분리 운영하는 발상 자체는 가능
- 기승전결(시작→핵심→보완→마무리) 흐름, 구체 행동 문장, 시작/종료 명확성은 일반 템플릿 품질 규칙과 동일하게 중요
