# ADHD 타이머 앱 MVP PRD

- 문서 버전: v1.0
- 작성일: 2026-02-26
- 문서 목적: MVP 개발을 위한 제품 요구사항(기능/수용기준/출시 기준) 정의
- 대상: Product, Design, Frontend, Backend, QA

## 1. 제품 개요

### 1.1 배경
ADHD 사용자는 시간 흐름 인지, 작업 시작, 마감 관리에서 반복적인 실행 장벽을 경험한다. 본 제품은 "시간을 보이게" 하고 "지금 해야 할 일"을 빠르게 선택하도록 지원하는 실행 중심 도구를 목표로 한다.

### 1.2 제품 한 줄 정의
원형 시각화 타이머와 우선순위 기반 일정관리, 마감 진행률, 외부 캘린더 연동을 결합한 ADHD 친화형 일정 실행 앱.

## 2. 목표 및 비목표

### 2.1 목표 (MVP)
- 사용자가 하루 일정 중 "지금 시작할 작업"을 10초 내 선택할 수 있다.
- 시각화 타이머로 집중 세션을 시작/유지/종료할 수 있다.
- 일정별 마감 압박도를 Progress Bar로 즉시 파악할 수 있다.
- 구글/애플 캘린더와 일정 동기화가 동작한다.

### 2.2 비목표 (MVP 제외)
- 팀 협업(공유 보드, 권한, 코멘트)
- AI 자동 일정 추천
- 고급 리포팅(월간 인사이트, 행동 코칭)

## 3. 타겟 사용자 및 핵심 시나리오

### 3.1 타겟 사용자
- 1차: 성인 ADHD 사용자(직장인/학생)
- 2차: 집중시간 관리가 필요한 일반 사용자

### 3.2 핵심 시나리오
1. 사용자는 오늘 해야 할 일을 등록한다.
2. 각 일정에 중요도와 우선순위를 지정한다.
3. 선택한 일정과 함께 뽀모도로 타이머를 시작한다.
4. 대시보드에서 마감 Progress Bar를 보고 위험 일정을 먼저 처리한다.
5. 외부 캘린더와 동기화해 기존 일정과 충돌 여부를 확인한다.

## 4. MVP 범위

### 4.1 In Scope
- 시각화 뽀모도로 타이머
- 일정 CRUD
- 중요도/우선순위 독립 관리
- 마감 남은시간 Progress Bar
- 구글/애플 캘린더 연동 및 수동 동기화

### 4.2 Out of Scope
- 다중 사용자 협업
- 반복 규칙의 고급 예외 처리(예: 매월 마지막 평일)
- 음성 입력/음성 알림

## 5. 기능 요구사항 (FR) 및 수용기준 (AC)

## FR-01 시각화 타이머 (뽀모도로 + 원형 남은시간)
- 설명: 집중/휴식 세션을 원형 UI로 표시하고 시간 경과를 직관적으로 제공한다.
- 우선순위: P0

### 사용자 스토리
- As a user, I want to see remaining focus time as a circle so that I can feel time passing.

### 수용기준
1. 기본 프리셋 `집중 25분 / 휴식 5분`이 제공된다.
2. 사용자는 집중/휴식 시간을 각각 수정할 수 있다.
3. 타이머는 `시작/일시정지/재개/종료/다음세션` 동작을 지원한다.
4. 원형 진행 UI와 `mm:ss` 숫자가 항상 동기화된다.
5. 세션 완료 시 기록(시작/종료/집중시간/중단횟수)이 저장된다.

### 예외/오류 처리
- 앱 재진입 시 진행중 세션이 3초 이내 복원된다.
- 백그라운드 전환 시 실제 경과시간 기반으로 잔여시간을 보정한다.

## FR-02 일정 등록/관리
- 설명: 일정 생성/수정/삭제/완료 처리를 제공한다.
- 우선순위: P0

### 사용자 스토리
- As a user, I want to manage today’s tasks quickly so that I can start without planning fatigue.

### 수용기준
1. 일정 생성 시 필수값은 `제목, 마감일시`다.
2. 일정 목록에서 완료/미완료 토글이 가능하다.
3. 일정 수정/삭제가 2클릭 이내로 가능하다.
4. 목록 정렬은 `우선순위순, 마감임박순`을 제공한다.
5. 오늘/이번주 필터가 동작한다.

### 예외/오류 처리
- 마감일시가 현재시각보다 과거면 경고를 표시하고 저장을 제한한다.
- 네트워크 실패 시 마지막 변경은 재시도 큐에 보관된다.

## FR-03 중요도 / 우선순위 분리
- 설명: 중요도(영향도)와 우선순위(실행순서)를 분리 저장한다.
- 우선순위: P0

### 사용자 스토리
- As a user, I want to separate importance from execution priority so that I can make better task choices.

### 수용기준
1. 중요도는 `High/Medium/Low` 3단계다.
2. 우선순위는 `1~5` 정수다(1이 가장 우선).
3. 중요도와 우선순위는 독립 수정 가능하다.
4. 일정 카드에서 두 값이 동시에 표시된다.
5. 정렬 시 중요도/우선순위를 각각 기준으로 선택할 수 있다.

## FR-04 일정 마감 Progress Bar
- 설명: 일정 생성시점 대비 마감까지의 시간 소진률을 막대로 표시한다.
- 우선순위: P0

### 사용자 스토리
- As a user, I want a deadline progress bar so that I can detect risky tasks at a glance.

### 수용기준
1. 각 일정 카드에 Progress Bar가 표시된다.
2. 계산식은 `elapsedRatio = (now - createdAt) / (dueAt - createdAt)`를 사용하고 0~100% 범위로 고정한다.
3. 색상 규칙은 `0~59% 녹색, 60~84% 주황, 85~100% 빨강`이다.
4. 보조 텍스트 `마감까지 n일 n시간`이 함께 표시된다.
5. 마감 초과 시 상태를 `Overdue`로 표시한다.

### 예외/오류 처리
- `dueAt <= createdAt`인 잘못된 데이터는 100% + 경고 상태로 렌더링한다.

## FR-05 구글/애플 캘린더 연동
- 설명: 외부 캘린더 연결, 일정 불러오기, 수동 동기화를 제공한다.
- 우선순위: P0

### 사용자 스토리
- As a user, I want to sync with my existing calendar so that all schedules stay consistent.

### 수용기준
1. Google OAuth 2.0 연결/해제 가능.
2. Apple Calendar(iCloud) 연결/해제 가능.
3. 연결 후 1회 수동 동기화로 외부 일정이 앱에 반영된다.
4. 앱 일정을 외부 캘린더로 내보낼 수 있다.
5. 동기화 결과(성공/실패/충돌) 로그가 사용자에게 표시된다.

### 동기화 정책 (MVP)
- 기본 주기: 수동 동기화(버튼) + 백그라운드 15분
- 충돌 처리: 최신 수정 우선, 이전 값은 충돌 로그에 보존
- 시간대 기준: 서버 UTC 저장, UI는 사용자 로컬 타임존 렌더링

## 6. 비기능 요구사항 (NFR)

## NFR-01 성능
- 대시보드 최초 로딩 2초 이내(표준 네트워크 환경)
- 타이머 애니메이션 60fps 목표
- 일정 목록 500개 기준 스크롤 지연 체감 최소화

## NFR-02 안정성
- 동기화 실패 재시도(최대 3회, 지수 백오프)
- 주요 API 에러율 1% 이하 목표

## NFR-03 보안/개인정보
- OAuth 토큰 암호화 저장
- 민감정보 로그 마스킹
- 최소 권한 스코프 사용

## NFR-04 접근성
- 타이머 상태는 색상 외 텍스트로도 구분
- 키보드 접근(시작/일시정지/완료)
- 색약 사용자 고려 색상 대비 준수

## 7. UX 요구사항

- 첫 화면에서 오늘 일정 + 타이머 시작 버튼이 동시에 보여야 한다.
- 일정 생성은 30초 내 완료 가능한 입력량으로 제한한다.
- 타이머 진행 중에는 현재 연결된 일정 제목을 상단 고정 표시한다.
- 마감 임박(Progress 85% 이상) 일정은 리스트 상단 강조 노출한다.
